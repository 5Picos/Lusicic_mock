Sección 4 — Secuencia de construcción

Las 3 páginas prioritarias validan todos los patrones del sistema. El resto se construye más rápido sobre esa base.

Orden de construcción por etapas

0
Scaffolding — shell + mock data
Layout shell (sidebar accordion + topbar) lib/mock-data.ts con todos los tipos Sistema de colores en Tailwind config StatusBadge component
1
/dashboard — Valida alertas + cards tipo A
Primer contacto real con el sistema. Si las alertas se ven bien aquí, se ven bien en toda la app.
SummaryCard tipo A (alertas) Tabla de alertas de camiones Tabla de alertas de choferes Lógica de máquina de estados Ordenamiento por prioridad
2
/pedidos — Valida cards tipo C + Modal + Sheet
Primera página con formulario complejo. Valida ambos patrones de dialog.
SummaryCard tipo C (informativas) DataTable con toolbar Lifecycle badges (pendiente→facturado) Modal "Asignar" (camión + chofer) Sheet "Nuevo pedido" (cliente + localidad + lista + artículos)
3
/remitos — Valida panel ámbar + Sheet multi-sección
El Sheet más complejo del sistema (artículos + peajes). Si funciona aquí, funciona en todo.
Panel ámbar de pendientes Tabla de remitos existentes Sheet "Registrar remito" sección artículos (read-only) Sheet sección peajes (editable por fila) Lógica auto-generación de gastos al guardar
4
Maestros — catálogos y ABMs simples
/camiones /choferes /clientes /proveedores /articulos /localidades /precios /tipos-gasto /usuarios
5
Administración — facturación y cobros
/facturacion /cuenta-corriente /cobros /cheques /gastos Mantenimiento historial
6
Informes — vistas de solo lectura
/informes/gastos /informes/ingresos /informes/comparativo /informes/viajes /informes/peajes /informes/facturas-destino
